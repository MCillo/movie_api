Documentation Page of movie_api app.

Request URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return Data(Description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user /movies/[Movie] GET None A JSON object holding data about a single movie: containing movie ID, name, director, genre, description, image URL, featured(boolean). Example:
{
movieId: 1
title: The Shawshank Redemption
director: {
directorName: Frank Darabont
directorBio: Frank Darabont's bio
directorBirth: January 01, 1950
directorDeath: Null
}
genre: {
genreName: Drama
genreDescription: Description for Drama
}
movieImageURL: image for the movie
featured: false
}
Return Data about a genre (description) by name/title e.g., "Thriller" /movies/[Genre] GET None A JSON object holding data about the genre selected Example:
{
genreName: Drama
genreDescription: Description for Drama
}
Return data about a director (bio, birth year, death year) by name /movies/[Director] GET None A JSON object holding data about the director selected. Example:
{
directorName: Frank Darabont
directorBio: Frank Darabont's bio
directorBirth: January 01, 1950
directorDeath: Null
}
Allow new users to register /users POST A JSON object holding data about the user to add: Structured Like:
{
name: Jane Doe
userName: JaneUserDoe
favorites: Pulp Fiction
}
A text informing the user of successful registration
Allow users to update their user info (username) /users/[Username] PUT None A text informing the user of succesful update
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added - more on this later) /users/[Favorites] PUT None A text informing the user of a succesful movie addition
Allow users to remove a movie from their list of favorites (showing on a text that a movie has been removed more on this later) /users/[Favorites] DELETE None A text informing the user of a succesful movie removal
Allow existing users to deregister (showing only a text that a user email has been removed - more on this later) /users/[Username] DELETE None A text informing the user of succesful account deactivation
