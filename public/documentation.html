Documentation Page of movie_api app.

Movie_API URL Endpoints
Request URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return Data(Description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user /movies/[Movie] GET None A JSON object holding data about a single movie: containing movie ID, name, director, genre, description, image URL, featured(boolean) etc.,
Return Data about a genre (description) by name/title e.g., "Thriller" /movies/[Genre] GET None A JSON object holding data about the genre selected
Return data about a director (bio, birth year, death year) by name /movies/[Director] GET None A JSON object holding data about the director selected
Allow new users to register /users POST A JSON object holding data about the user to add: user name (email), user password, auto generated user id A text informing the user of successful registration
Allow users to update their user info (username) /users/[Username] UPDATE None A text informing the user of succesful update
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added - more on this later) /users/[Favorites] UPDATE None A text informing the user of a succesful movie addition
Allow users to remove a movie from their list of favorites (showing on a text that a movie has been removed more on this later) /users/[Favorites] REMOVE None A text informing the user of a succesful movie removal
Allow existing users to deregister (showing only a text that a user email has been removed - more on this later) /users/[Username] REMOVE None A test informing the user of succesful account deactivation
